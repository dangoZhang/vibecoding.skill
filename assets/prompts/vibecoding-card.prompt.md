# vibecoding.skill 分享卡 Prompt

用于生成 `vibecoding.skill` 的默认分享卡。

默认是中性科技感等级卡。修仙风格只作为彩蛋。

如果用户明确要求英文，或轨迹几乎全是英文，卡片整体切到英文。

## 目标

把真实的 Code Agent 协作记录，压成一张适合转发的 vibecoding 等级卡。

重点有三件事：

- 一眼看清等级
- 一眼看懂这轮能力判断
- 一眼知道下一步该怎么练

## 默认标题

- title：`vibecoding.skill`
- slogan：`蒸馏你的 Code Agent 记录`

## 默认版式

从上到下固定为：

1. title
2. slogan
3. `等级`
4. `vibecoding能力`
5. `判词`
6. `下一步`
7. `模型 | tokens`
8. `用户 | 时间`

要求：

- 默认卡片里不要出现“阶段”
- 默认卡片里不要出现“稳定期”“成形期”这类单独一列
- `等级` 是唯一超大字
- 其余所有文字使用统一字号
- 三个正文区块必须独立，边距统一，文字左对齐
- 底部两行信息只做补充，不抢正文

## 默认文案原则

- 先说人话
- 句子要短，要顺口
- 文案要像用户愿意直接发出去的总结
- `vibecoding能力` 写能力摘要，必须带上这轮最亮眼的地方和最该补的地方
- `判词` 只写这一等级的人通常能做到什么
- `下一步` 只写最值得先补的一步
- 生成后要自查一遍，删掉空话、套话和解释腔
- 英文模式下保留 `Level`、`Model`、`tokens` 这类常见写法

## 默认文案示例

- title：vibecoding.skill
- slogan：蒸馏你的 Code Agent 记录
- 等级：L6
- vibecoding能力：你已经会把一段具体工作先交给 AI 推进，再回来收方向、结果和风险。这轮最亮眼的是执行推进和结果验证：任务能继续往前走，收尾时也开始看证据。现在最该补的是上下文承接和协作节奏：回合一长容易散，关键边界还不够稳定地反复确认。
- 判词：这一层的人，会让 AI 先走一段，再回来收方向和结果。
- 下一步：下一层要看的，是能不能把多 Agent、多工具真正组织起来。下一轮先把任务拆成短回合，每两轮重申一次主线和边界。

## 视觉要求

- 更像等级卡，不像长报告截图
- 外层背景使用等级色系
- 内层主卡使用深色面板
- 不要额外加一列“阶段”
- 不要堆无意义的图案和色块
- 每个区块边框、圆角、留白一致
- 所有正文左对齐
- 最后一行文字和底部信息之间必须留出完整安全边距

## 彩蛋模式

用户明确要求修仙版时，再切成：

- title：`vibecoding.skill`
- slogan：`蒸馏你的 Code Agent 记录`
- 主字：`境界 | 等级`

修仙版也要保留：

- 模型
- tokens
- 用户
- 时间

修仙词只出现在标题、判词和少量正文里。
